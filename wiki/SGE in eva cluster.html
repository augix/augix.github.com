SGE in eva cluster

Profile

User private default request files are processed under the locations $HOME/.sge_request and $cwd/.sge_request. The working directory local default request file has the highest precedence

E-mail

send email when the job ends: -m e -M your@email

sge script

job.sge <bash>

  1. IMPORTANT: specifies the interpreting shell for this job to be the Bash shell

#$ -S /bin/sh

  1. execute the job for the current working directory.

#$ -cwd

  1. set the required cpu time of your job
  2. HH:MM:SS

#$ -l h_cpu=20:00:00

  1. set the required memory:
  2. normal:3.7 GB, 72 h, 4 slots per host.
  3. bigmem: 7.7 GB, 72 h, 2 slots per host.
  4. hugemem: 15.7 GB, 72 h, 1 slot per host.

#$ -l h_vmem=7000M

  1. don't flood myself with e-mail

#$ -m n

  1. join the error and standard output streams

#$ -j yes

  1. Defines or redefines the path used for the standard error stream
  2. and the standard output stream of the job.

#$ -o ./sge.out/ #$ -e ./sge.out/

  1. name of the job

#$ -N mapping_simulation

  1. all environment variables active within the qsub utility be exported to the context of the job.

#$ -V

  1. If the value of -b is 'y', then command may be a binary or script. If the value of -b is 'n' then command needs to be a script and it will be handled as script.

#$ -b y

echo $1 | ./job.Rf </bash>

submit script

<bash> for i in seq 1 100; do qsub job.sge $i; done </bash>

Other tips

SGE command

  • submit jobs qsub
  • check the state of jobs qstat
  • check the state of a particular job qstat -j JOB_ID
  • check the state of severs qstat -g c
  • delete jobs qdel -u augix
